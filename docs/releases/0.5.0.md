# FLOWnote 0.5.0

0.5.0 是一次大版本更新：FLOWnote 从「主要依赖 OpenCode 桥接」升级为「Obsidian 内置 AI 工作流 + 可选 OpenCode 桥接」的双模式架构。

## 更新亮点

- 新安装用户默认使用内置直连 API 模式；已有用户升级后会保留原来的 OpenCode 桥接模式，避免更新后工作流突然变化。
- 新增首次安装/升级引导弹窗，解释内置 AI 模式与 OpenCode 桥接模式的差异，并提供设置入口。
- 直连 API 模式现在具备 Obsidian 原生工具链能力，包括笔记读取、写入、搜索、每日笔记、任务、标签、反链、属性、文件移动、目录创建等。
- 支持导入完整的外部 skill 文件夹，不再只编辑单个 `SKILL.md`；技能管理页可以读取、编辑、删除、导入自定义 skills。
- 新增 `web_request` 工具，支持 POST、Authorization header、JSON body 和 `$SECRET` 占位符，可用于微信读书等官方 skills。
- 新增严格询问、仅危险操作询问、全自动三种工具权限模式；全自动模式会显示风险提示。
- 移动端技能列表、模型选择器浅色模式、推荐卡片、AI Provider 路径等做了适配；桌面端也补齐了推荐卡片。
- AI 回复中的 Obsidian 笔记路径现在会自动识别为可点击链接，点击即可打开对应笔记。
- 修复设置页切换 OpenCode 桥接模式后自动跳回默认模式的问题。
- 修复服务商、模型等选择后回跳的问题，并补强设置组件取值逻辑。
- 修复模型选择器外框、移动端浅色模式样式、插件 logo、消息复制按钮、滚动导航等一批 UI 细节。
- 补充 README 隐私、联网、Vault 访问、剪贴板、本地环境读取说明；OpenCode 子进程只转发白名单环境变量。
- 补充直连 Agent、权限策略、skills 导入、模板管理、路径链接、移动端入口等测试；当前自动化测试覆盖 757 个用例。

## 升级提示

- 如果你已经在使用 OpenCode 桥接模式，升级后会继续保留该模式。
- 如果你是新用户，默认会进入内置直连 API 模式，需要在 FLOWnote 设置中配置 AI 服务商 API Key。
- 移动端使用内置 AI Provider 路径，不能从移动端沙盒启动桌面 OpenCode CLI。

# FLOWnote 0.5.0

0.5.0 is a major release that moves FLOWnote from an OpenCode-first bridge into a dual-mode architecture: built-in Obsidian-native AI workflows plus an optional OpenCode bridge.

## Highlights

- New installs use built-in direct API mode by default. Existing users keep their previous OpenCode bridge mode after upgrading.
- A new startup notice explains the difference between built-in AI mode and OpenCode bridge mode, with a direct path into settings.
- Direct API mode now includes Obsidian-native tools for reading, writing, searching, daily notes, tasks, tags, backlinks, properties, file moves, and folder creation.
- You can import complete external skill folders, not only edit a single `SKILL.md`. The skill management settings can list, edit, delete, and import custom skills.
- Added `web_request` with POST, Authorization headers, JSON bodies, and `$SECRET` placeholders for API-backed skills such as WeRead.
- Added strict ask, dangerous-only ask, and full-auto permission modes. Full-auto mode includes an explicit risk warning.
- Improved mobile skill visibility, light-mode model selector styling, recommendation cards, and built-in provider routing. Desktop now also gets recommendation cards.
- Obsidian vault paths in AI replies are detected and rendered as clickable links.
- Fixed settings mode persistence so the OpenCode bridge no longer jumps back to default mode.
- Provider and model selections now persist correctly across settings refreshes.
- Fixed model selector outlines, mobile light mode colors, plugin logo, copy buttons, scroll navigation, and related interface details.
- README disclosures now cover networking, vault access, clipboard use, and local environment access. Optional OpenCode child processes receive only allowlisted environment values.
- Added tests for the direct agent runtime, permission policy, skill import, template management, vault path links, and mobile entry points. The current automated suite covers 757 tests.

## Upgrade Notes

- If you already use OpenCode bridge mode, FLOWnote keeps that mode after the update.
- If you are a new user, FLOWnote defaults to built-in direct API mode and requires an AI provider API key in settings.
- Mobile uses FLOWnote's built-in AI provider path and cannot launch a desktop OpenCode CLI from the mobile sandbox.

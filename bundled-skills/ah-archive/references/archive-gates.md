# Archive Gates (W -> F/L)

项目归档前必须通过以下门禁，确保“项目结束不是终点，而是沉淀起点”。

## Gate 1: Completion Gate

至少满足 2 项：

- 成功标准已达成或明确放弃并记录原因。
- 关键交付物已产出并可追溯。
- 主要任务已清空，剩余项已移交或取消。

未通过时：

- 不归档。
- 在项目状态写 `阻塞:<原因>`。
- 返回一个最小补齐清单（最多 3 项）。

## Gate 2: Knowledge Gate

必须产出至少 1 条“可复用经验”并归类为：

- 决策经验（为什么这么做）。
- 执行经验（如何做更稳）。
- 反模式（下次不要怎么做）。

如果三类都为空：

- 允许归档，但标记“低沉淀项目”。

## Gate 3: Handoff Gate

判断是否需要交接给 `ah-card`：

- 若存在可迁移洞见，写入 `待交接:ah-card`。
- 若无可迁移洞见，直接标记项目 `已完成`。

## Gate 4: Recoverability Gate

归档后必须能回答：

- 项目为什么开始？
- 项目如何推进？
- 项目为何结束？
- 有哪些经验可复用？

如果无法回答，补写归档复盘摘要再结束。

# ah-archive

做项目收尾：复盘、提炼、归档、更新索引。

## 必须遵守

- Memory 默认最小读取：`STATUS + target project summary`。
- 归档前先确认项目状态与关键产出。
- 项目链接始终指向 `📍 项目总览.md`。

## 流程

1. 确认要归档的项目。
2. 检查完成度与未收口事项。
3. 引导项目复盘（成果/收获/复用点/遗留）。
4. 生成归档摘要并移动到归档目录。
5. 更新领域页、`STATUS.md`、`index.json`。

## 按需读取 References

- 归档前置检查与复盘模板：`references/archive-playbook.md`

## 输出

- 归档路径。
- 复盘摘要。
- 可转卡片候选（可选，交给 `ah-card`）。

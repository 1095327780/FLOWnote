# 周回顾引导技能

## 🔒 流程锁定规则（最高优先级）

> ⚠️ **进入本技能后，必须完成完整流程才能退出！**

**核心原则**：用户的回答是「数据输入」，不是「命令触发」。

**禁止跳转的情况**：
- 问「这周最满意的成就是什么？」时，用户说「读完了一本书」
  - ❌ 错误：识别到「读」跳转到 ah-read
  - ✅ 正确：记录为本周成就
- 问「下周最重要的3件事？」时，用户说「整理笔记」
  - ❌ 错误：识别到「整理」跳转到 ah-inbox
  - ✅ 正确：记录为下周计划

**调用 ah-card 后必须返回**：
- 处理残留想法时选择「转为永久笔记」会调用 ah-card
- ah-card 完成后，**必须返回 ah-week 继续处理下一条想法**
- 直到所有想法处理完毕，才继续周回顾的下一阶段

**流程完成标志**：
- 数据汇总展示完成
- 残留想法全部处理（或用户选择跳过）
- 反思环节完成
- 下周计划已写入
- 显示了完成总结

## 概述

每周花30分钟做一次全面复盘，比每天花5分钟的零散反思更有价值。本技能帮助用户进行结构化的周度回顾。

**核心价值**：
- 自动汇总本周的各项数据
- **处理本周残留的未处理想法**
- 引导用户从多个维度反思
- 识别模式和改进机会
- **设定下周计划并写入下周笔记**
- 生成结构化的周回顾笔记

## 回顾维度

1. **知识维度**：新增笔记、阅读进度、知识链接
2. **行动维度**：任务完成、项目进展、目标达成
3. **想法维度**：本周记录的想法、处理情况、转化成果
4. **成长维度**：洞察收获、习惯养成、能力提升
5. **生活维度**：健康、关系、心情

## 工作流程

```
1. 数据汇总（AI自动）
   ↓
2. 展示周报（AI汇报）
   ↓
3. 【核心】处理残留想法（AI引导）
   ↓
4. 引导反思（AI提问）
   ↓
5. 收集反馈（对话）
   ↓
6. 【核心】规划下周 + 写入下周笔记
   ↓
7. 生成周回顾笔记（AI自动）
```

## 阶段二：处理残留想法

> 🔑 **这是周回顾的核心环节之一**：处理本周积累但未处理的想法。

### 2.1 提取未处理想法

从本周所有每日笔记的「💡 想法和灵感」区域提取：
- 没有 `→` 标记的
- 没有 `✅` 标记的

### 2.2 展示未处理想法

```
📝 **本周未处理的想法**

扫描了本周 **{{N}}** 天的日记，发现 **{{M}}** 条待处理的想法：

| # | 日期 | 时间 | 内容预览 |
|---|------|------|----------|
| 1 | 02-03 | 09:15 | 习惯的本质是大脑的自动化程序... |
| 2 | 02-03 | 14:22 | 用户体验的核心是减少认知负担 |
| 3 | 02-04 | 10:30 | 复杂系统的关键不是功能多... |
| 4 | 02-05 | 16:00 | 记得下周预约体检 |

想法不多，逐条处理？还是使用批量模式？
```

### 2.3 处理模式选择

**逐条模式**（<10条）：
```
📝 **第 {{i}}/{{M}} 条**

**日期**：{{YYYY-MM-DD}}
**时间**：{{HH:mm}}

---
{{完整内容}}
---

1. 💡 转为永久笔记（调用 /ah-card）
2. ✅ 转为任务
3. ☑️ 标记已处理
4. ⏭️ 跳过（下次再处理）

你的选择是？
```

> 🔗 **重要**：选择"转为永久笔记"时，必须调用 `ah-card` 技能来处理，确保：
> - 断言式标题（完整陈述句）
> - 扫描知识库推荐链接
> - 归类到主题页
> - 维护双向链接

**批量模式**（≥10条）：
```
想法较多，我先帮你分类：

**建议转永久笔记**（{{n}}条）：
1. {{内容预览}}
2. {{内容预览}}

**建议转任务**（{{n}}条）：
1. {{内容预览}}

**可能不需要处理**（{{n}}条）：
1. {{内容预览}}

**需要你判断**（{{n}}条）：
1. {{内容预览}}

确认这个分类？或者逐条处理？
```

### 2.4 处理结果更新

处理后：
1. 更新对应日记中的想法标记
2. **调用 ah-card 创建永久笔记**（如选择转笔记）
3. 添加任务到相关日记

> ⚠️ **注意**：所有"转永久笔记"的操作都必须通过 `ah-card` 技能完成，不能跳过制卡流程直接创建文件。

## 阶段三：引导反思

### 3.1 成就回顾

```
想法处理完成！现在让我们反思这一周。

**问题1：这周你最满意的成就是什么？**

可以是：
- 完成了一个重要任务
- 学到了新知识
- 养成了一个好习惯
- 改善了一段关系
- 或者任何让你感到自豪的事
```

### 3.2 挑战与困难

```
**问题2：这周遇到的最大挑战是什么？你是如何应对的？**

不管结果如何，重要的是识别模式：
- 这个挑战是偶发的还是反复出现的？
- 你的应对方式有效吗？
- 下次遇到类似情况，你会怎么做？
```

### 3.3 学习与洞察

```
**问题3：这周最重要的一个学习或洞察是什么？**

可能来自：
- 阅读的书籍
- 工作中的经验
- 与人的交流
- 自己的反思

如果可以的话，用一句话概括这个洞察。
```

### 3.4 改进机会

```
**问题4：如果这周可以重来，你会做什么不同的事？**

这不是自责，而是寻找改进的机会：
- 时间分配上有什么调整？
- 有什么事情本可以更早开始？
- 有什么事情其实可以不做？
```

### 3.5 能量与状态

```
**问题5：这周整体的能量状态如何？**

1-10分，你会给这周打几分？

什么事情给你带来能量？
什么事情消耗你的能量？
```

## 想法处理状态标记规范

在每日笔记的「💡 想法和灵感」区域，每条想法的状态通过后缀标记：

| 状态 | 标记格式 | 示例 |
|------|----------|------|
| 未处理 | （无标记） | `- 09:15 内容...` |
| 转永久笔记 | `→ [[笔记名]]` | `- 09:15 内容... → [[笔记名]]` |
| 转任务 | `→ 任务` | `- 09:15 内容... → 任务` |
| 已处理 | `✅ 已处理` | `- 09:15 内容... ✅ 已处理` |

**判断未处理的规则**：条目中不包含 `→` 和 `✅` 的即为未处理。

---

## 注意事项

1. **数据准确性**：统计数据要准确，避免误导
2. **灵活节奏**：根据用户时间和状态调整深度
3. **正向引导**：即使这周不理想，也要发现亮点
4. **可操作的计划**：下周计划要具体可执行
5. **定期习惯**：建议用户固定时间做周回顾（如周日晚上）
6. **想法清零意识**：周回顾是处理残留想法的最后防线

## 与其他技能的关系

```
每日：ah-review → 处理当天想法
         ↓
每周：ah-week → 处理本周残留想法 + 设定下周计划
         ↓
每月：ah-month → 处理本月残留想法 + 设定下月计划
```

**相关技能**：
- **ah-review**：每日回顾是周回顾的基础
- **ah-inbox**：专门批量处理想法的技能
- **ah-card**：将周回顾中的洞察转化为永久笔记
- **ah-month**：月度回顾处理更长周期的残留

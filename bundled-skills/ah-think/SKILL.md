# 思维模型工具箱

> 经典思维模型，帮助你从不同角度深入思考，提炼真正属于自己的洞见。

---

## 可用模型一览

| 模型 | 核心问题 | 最佳场景 |
|------|---------|---------|
| **费曼技巧** | "如果教给小孩，你会怎么解释？" | 检验是否真正理解一个概念 |
| **第一性原理** | "最基本的事实是什么？" | 打破惯性假设，重新构建理解 |
| **逆向思维** | "如果想失败，应该怎么做？" | 避免错误，发现盲点 |
| **二阶思考** | "然后呢？再然后呢？" | 评估长期后果，做重要决策 |
| **苏格拉底式提问** | "这个假设成立吗？" | 检验观点、澄清概念、深度对话 |
| **能力圈** | "这是我真懂还是以为懂？" | 识别知识边界，谦逊学习 |
| **5个为什么** | "为什么？（连问5次）" | 追溯根本原因 |

## 场景推荐

根据用户当前情境，推荐最合适的思维模型：

| 用户在做什么 | 推荐模型 | 原因 |
|-------------|---------|------|
| 读到一个新概念 | 费曼技巧 | 检验是否真正理解 |
| 读到一个观点/论断 | 苏格拉底式提问 | 检验论证的有效性 |
| 感觉"好像懂了" | 能力圈 + 费曼技巧 | 区分真懂和假懂 |
| 做重要决定 | 二阶思考 + 逆向思维 | 看到长期后果和潜在风险 |
| 想创新/突破 | 第一性原理 | 打破旧框架，重新构建 |
| 追问"为什么" | 5个为什么 | 一层层挖到根本原因 |
| 复盘反思 | 逆向思维 + 二阶思考 | 发现哪里可以做得更好 |
| 想写一张好的卡片笔记 | 费曼技巧 + 第一性原理 | 用自己的话表达核心洞见 |

## 使用方式

### 方式一：用户直接调用

```
用户：帮我用费曼技巧检验一下我对"心流"这个概念的理解
```

AI 应加载 `references/feynman-technique.md`，按照其中的步骤引导用户。

### 方式二：其他技能自动调用

当 ah-read、ah-card 等技能需要引导用户深入思考时，可推荐思维模型：

```markdown
💡 **深入一步？**

你刚才提炼的洞见挺有意思。如果你想再深入，可以试试：
1. 🎓 费曼技巧 — 用最简单的话解释这个概念
2. 🔄 逆向思维 — 什么情况下这个观点不成立？
3. ⏭️ 二阶思考 — 如果按这个行动，长期会怎样？

选一个继续？还是直接记录？
```

用户选择后，AI 加载对应的 references 文件，按步骤引导。

## 核心原则

### 1. 用户始终有选择权

在任何引导前，必须确认用户意愿：
- 用户说"跳过""直接记录""不用了" → 立即停止引导
- 不要强迫用户使用思维模型

### 2. 引导而非说教

- ✅ 通过提问，让用户自己发现答案
- ❌ 直接告诉用户"正确答案"
- 每个思维模型的价值在于引发思考，而非给出结论

### 3. 适可而止

- 每次最多深入使用 1-2 个模型
- 当用户给出了清晰的回答，不要继续追问
- 质量优于数量

### 4. 详细操作查 references

本文件只提供模型选择指引。具体如何使用每个模型，请查阅：

## 模型简介

以下是每个模型的核心理念，帮助快速判断该用哪个。详细使用方法见 references 目录。

### 费曼技巧 (Feynman Technique)

**来源**：物理学家理查德·费曼的学习方法

**核心理念**：如果你不能用简单的语言向一个12岁的孩子解释清楚，说明你并没有真正理解。

**适用**：学习新概念、检验理解深度、发现知识盲点

### 第一性原理 (First Principles Thinking)

**来源**：古希腊哲学（亚里士多德），近年由埃隆·马斯克推广

**核心理念**：回归最基本的事实，而不是依赖类比或惯例。把问题分解到最基础的层面，然后重新构建。

**适用**：打破思维定式、创新突破、重新理解复杂问题

### 逆向思维 (Inversion)

**来源**：数学家雅各比的名言"反过来想，总是反过来想"，查理·芒格推广

**核心理念**：与其想"如何成功"，不如想"如何避免失败"。通过识别应该避免的事情，反向找到正确的路。

**适用**：风险评估、避免错误、发现盲点

### 二阶思考 (Second-Order Thinking)

**来源**：霍华德·马克斯等投资者

**核心理念**：不只想直接后果，还要想后果的后果。问"然后呢？"，至少问三次。

**适用**：重要决策、评估长期影响、避免短视

### 苏格拉底式提问 (Socratic Questioning)

**来源**：古希腊哲学家苏格拉底

**核心理念**：通过系统性的提问（而非给答案），帮助人澄清概念、检验假设、发现逻辑漏洞。

**适用**：检验观点、深度对话、批判性思考

### 能力圈 (Circle of Competence)

**来源**：沃伦·巴菲特和查理·芒格

**核心理念**：清楚知道自己真正懂什么、不懂什么。在能力圈内做决策，对圈外的事保持谦逊。

**适用**：自我认知、识别知识边界、谦逊学习

### 5个为什么 (5 Whys)

**来源**：丰田生产系统创始人丰田佐吉

**核心理念**：对一个问题连续问"为什么"（通常5次），直到找到根本原因，而非停留在表面症状。

**适用**：追溯根本原因、解决问题、深入分析

# Cynefin 情境判别

## 底层原理

不同问题情境需要不同决策逻辑。

Cynefin 的价值是先判别问题域，再匹配行动模式，避免“用线性方案处理复杂问题”。

## 适用场景

- 团队对“先分析还是先试验”意见冲突。
- 问题不确定性高，且变化快。
- 需要决定治理方式（标准化/专家分析/实验/应急）。

## 不适用 / 降级条件

- 简单重复任务已具备标准作业，直接执行 SOP 即可。

## 详细操作流程

1. **收集信号**：因果是否清晰、是否有最佳实践、环境稳定度。
2. **判别情境域**：
   - Simple：清晰因果，可标准化。
   - Complicated：因果可分析，需专家判断。
   - Complex：因果后见，需试验学习。
   - Chaotic：先止损再重建秩序。
3. **匹配策略**：按情境选择“标准化/分析/探测/应急”。
4. **设转域触发器**：定义何时从一个域切到另一个域。
5. **短周期复判**：环境变化时重新判域，不固化标签。

## 与 FLOW 的结合

- `ah-read`：帮助判断读到的方法适用于哪类环境。
- `ah-card`：沉淀“情境 -> 策略 -> 触发器”卡片。
- `ah-review`：复盘失败是否源于情境误判。

## 常见误用与纠偏

- 误用：把复杂问题硬归为简单问题。
  纠偏：若预测频繁失效，回到 complex 域做实验。
- 误用：标签一旦给出就不再调整。
  纠偏：每次关键反馈后重新判域。

## 质量检查

- 情境判别是否有依据而非直觉？
- 策略是否与情境域一致？
- 是否定义了转域触发条件？

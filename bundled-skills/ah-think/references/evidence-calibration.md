# 证据校准 (Claim-Evidence-Reasoning)

## 底层原理

高质量判断必须把“主张-证据-推理”显式拆开。

结论可靠性取决于证据质量与推理链完整度，而非表达自信度。

## 适用场景

- 需要判断观点可信度。
- 团队讨论常出现“观点先行、证据补写”。
- 制卡时需要可追溯论证。

## 不适用 / 降级条件

- 紧急执行场景可先做轻量 CER（1 条主张 + 1 条关键证据）。

## 详细操作流程

1. **定义主张**：写成可验证命题，避免空泛判断。
2. **证据盘点**：列出支持/反对证据，标注来源与时效。
3. **证据分级**：区分一手数据、二手报告、个人经验、推测。
4. **推理建链**：写清证据如何推到主张，中间假设是什么。
5. **缺口识别**：找证据空白与逻辑跳步。
6. **置信更新**：根据证据质量更新结论置信度。
7. **下一步取证**：给出最小补证动作。

## 与 FLOW 的结合

- `ah-read`：将阅读观点转换为可审计论证链。
- `ah-card`：沉淀“主张-证据等级-边界”卡片。
- `ah-review`：复盘决策失误是否源于证据质量不足。

## 常见误用与纠偏

- 误用：引用数量多但质量低。
  纠偏：优先高质量少量证据，不堆叠弱证据。
- 误用：证据与主张不对应。
  纠偏：每条证据必须回答“它支持哪一步推理”。

## 质量检查

- 主张是否可验证？
- 证据来源和等级是否清晰？
- 推理链是否存在未说明的跳步？

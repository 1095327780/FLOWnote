# 继承学习与创业学习

## 底层原理

- 继承学习：复用前人已验证模型，追求效率。
- 创业学习：在未知环境中自建模型，追求创新与适配。

现实中通常先继承，再在失配点创业。

## 适用边界

### 继承优先

- 问题有成熟方法与公开材料。
- 时间/资源有限，需快速达标。

### 创业优先

- 面对新问题、旧模型失效、前沿探索。
- 环境变化快，通用答案不可用。

## 详细操作流程

1. **模式选择**：判断问题成熟度与资料可得性。
2. **继承路径**：获取材料 -> 重建概念 -> 重建映射 -> 场景验证。
3. **创业路径**：采集数据 -> 提出假设 -> 小实验验证 -> 迭代模型。
4. **混合策略**：继承 70%，在关键失配点创业 30%。
5. **退出条件**：当创业模型稳定后沉淀为可继承资产。

## 反模式

- 记言代学：背别人结论但不重建模型。
- 创业冲动：明明有成熟解法仍从零试错。
- 只创业不沉淀，导致无法复用。

## 输出模板

- `learning_mode`: inheritance / pioneering / hybrid
- `validation_path`
- `experiment_plan`
- `codification_plan`

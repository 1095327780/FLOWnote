# 预演失败 (Pre-mortem)

## 底层原理

预演失败通过“事前回看失败”降低乐观偏差，让团队在执行前就暴露脆弱点。

关键价值：在成本最低时修正计划。

## 适用场景

- 新项目启动、重大决策前。
- 多人协作、责任边界复杂。
- 对外承诺后回滚代价高。

## 不适用 / 降级条件

- 低风险短任务，可降级为“Top 3 风险快速检查”。

## 详细操作流程

1. **设定失败时间点**：例如“3 个月后项目失败”。
2. **独立列因**：先个人书写失败原因，避免群体从众。
3. **合并归类**：按需求、资源、执行、协作、外部变量分组。
4. **风险排序**：按影响 x 可控性选出前 2-3 个关键风险。
5. **设计防线**：每个风险配预防动作、应急动作、责任人。
6. **设置预警**：定义最早可观测的异常信号。
7. **嵌回计划**：把防线与预警写入里程碑检查点。

## 与 FLOW 的结合

- `ah-read`：把策略性建议转为可执行风险清单。
- `ah-card`：沉淀“风险模式 -> 防线模板”。
- `ah-review`：对照预警信号复盘“是否提前发现”。

## 常见误用与纠偏

- 误用：讨论风险但不落实责任。
  纠偏：每条关键风险必须绑定责任人与触发条件。
- 误用：只做一次会议。
  纠偏：在关键里程碑重复执行轻量版预演。

## 质量检查

- 是否覆盖了高影响失败路径？
- 是否定义了前置预警信号？
- 是否有清晰责任人与动作？

# 记忆编码与框架选择

## 底层原理

记忆不是照相存储，而是“现象 -> 概念框架 -> 可调用表征”的编码过程。

框架选错，会导致“听懂了词、没抓住事”。

## 适用场景

- 被对话框架套牢，无法还原事实。
- 信息量过大，记不住、说不清。
- 同一事实在不同叙事里结论冲突。

## 详细操作流程

1. **编码回撤**：把概念标签临时拿掉，重述裸事实。
2. **识别旧框架**：当前是用什么概念在解释事实？
3. **比较候选框架**：哪个框架信息损失最小、行动价值最高？
4. **重编码**：用新框架重述问题与目标。
5. **组块化**：把碎片概念打包为可调用单元。
6. **回测**：新框架是否提升理解、记忆与决策速度？

## 反模式

- 把对方用词当事实本身。
- 只做语义替换，不改行动结构。
- 为了压缩记忆而过度丢失关键细节。

## 输出模板

- `raw_facts`
- `old_frame`
- `new_frame`
- `chunk_plan`
- `action_delta`

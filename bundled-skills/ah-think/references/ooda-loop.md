# OODA 循环

## 底层原理

OODA（Observe-Orient-Decide-Act）用于高不确定环境下的快速迭代决策。

关键不在“快做决定”，而在“快速感知-快速校准”。

## 适用场景

- 环境变化快、信息持续更新。
- 需要边做边学的执行任务。
- 决策可逆，且反馈可快速获取。

## 不适用 / 降级条件

- 高不可逆决策需先补风险分析，不应只靠 OODA 快速循环。

## 详细操作流程

1. **Observe**：收集最新关键信号与异常。
2. **Orient**：用当前模型解释信号，标记不确定点。
3. **Decide**：选择最小可行且可逆的下一步动作。
4. **Act**：执行并记录结果，不追求一次最优。
5. **Loop**：基于新反馈立即进入下一轮。

节奏建议：
- 每轮只追一个核心假设。
- 每轮都定义“成功/失败”判据与退出条件。

## 与 FLOW 的结合

- `ah-read`：用于动态主题的连续学习与判断更新。
- `ah-card`：沉淀“信号 -> 判断 -> 动作 -> 反馈”链路卡。
- `ah-review`：复盘循环速度与判断质量是否同步提升。

## 常见误用与纠偏

- 误用：只强调速度，忽略 Orient 质量。
  纠偏：每轮至少写出一个判断依据。
- 误用：动作不可逆仍快速推进。
  纠偏：高风险动作前补风险与边界分析。

## 质量检查

- 每轮是否有清晰观测信号？
- 决策是否保持可逆与可验证？
- 是否真实根据反馈调整了下一轮？

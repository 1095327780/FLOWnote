# 二阶思考 (Second-Order Thinking)

## 底层原理

一阶思考看“立即结果”，二阶思考看“结果引发的后续连锁反应”。

关键是时间维度和系统反馈，而不是把链条拉得越长越好。

## 适用场景

- 有明显短期收益但长期不确定的决策。
- 多方博弈、激励变化、资源再分配场景。
- 策略可能改变他人行为时。

## 不适用 / 降级条件

- 可逆小决策、低影响事项，用一阶判断即可。
- 信息太少时，先做小实验再扩展到二阶分析。

## 详细操作流程

1. **定义行动 A**：不讨论泛泛目标，只分析具体动作。
2. **列一阶效应**：立即收益、立即成本、受影响对象。
3. **追二阶链条**：至少写出 2 条“然后呢”路径。
4. **标注时间窗**：短期/中期/长期分别可能出现什么。
5. **识别反馈环**：是否形成强化环或抑制环。
6. **比较备选动作**：对比“总效益 + 波动 + 不可逆风险”。
7. **给出触发器**：定义何时切换策略。

可选追问（按需）：
- 若所有人都采取同样策略，会发生什么？
- 二阶副作用何时超过一阶收益？
- 哪个指标能最早显示链条已偏离预期？

## 与 FLOW 的结合

- `ah-read`：筛出“短期正确、长期可疑”的洞见。
- `ah-card`：制作“行动 -> 一阶 -> 二阶 -> 风险”决策卡。
- `ah-review`：复盘是否只优化了短期指标。

## 常见误用与纠偏

- 误用：无限外推导致分析瘫痪。
  纠偏：最多到三阶，并绑定决策时窗。
- 误用：把推测当确定性预测。
  纠偏：输出必须附带置信度与监测点。

## 质量检查

- 是否明确了一阶收益与二阶代价？
- 是否识别关键反馈环与时间窗？
- 是否提供策略切换触发条件？

# 不确定性校准 (Uncertainty Calibration)

## 底层原理

不确定性校准不是消除不确定，而是把它显式化并用于决策设计。

核心动作：给结论配置信心区间、触发条件和可逆动作。

## 适用场景

- 信息不足但必须行动。
- 结果波动大、难以一次判断正确。
- 需要向他人解释“为什么现在这么做”。

## 不适用 / 降级条件

- 证据充足且风险低的标准任务，可采用固定流程。

## 详细操作流程

1. **分层不确定性**：区分已知、未知、关键未知。
2. **给出置信度**：高/中/低并写明依据。
3. **定义触发器**：哪些新信息会改变当前结论。
4. **设计可逆动作**：先做低后悔、可回滚的小步行动。
5. **设置监测频率**：何时复查、用什么信号更新判断。
6. **更新结论**：触发条件满足时立即重算策略。

## 与 FLOW 的结合

- `ah-read`：避免把阅读结论当成绝对真理。
- `ah-card`：卡片中保留“置信度 + 更新触发器”。
- `ah-review`：复盘“错在判断”还是“错在不更新”。

## 常见误用与纠偏

- 误用：用“有不确定性”当不行动借口。
  纠偏：不确定时优先可逆实验而非停滞。
- 误用：置信度标注后不再更新。
  纠偏：必须绑定触发器与复查窗口。

## 质量检查

- 是否给出了明确置信度与理由？
- 是否定义了会改判的触发条件？
- 当前动作是否可逆且代价可控？

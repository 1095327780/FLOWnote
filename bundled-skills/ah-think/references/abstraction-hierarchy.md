# 抽象层级与下上结构

## 底层原理

知识应用不是“越抽象越好”，而是要在对象层与共相层之间形成可往返映射。

- 下层：具体对象、现象、任务。
- 上层：概念、规律、策略。

有效思考是“上层指导下层、下层校验上层”。

## 适用场景

- 感觉观点正确但无落地动作。
- 感觉案例很多但总结不出规律。
- 讨论中出现“太空”或“太碎”。

## 详细操作流程

1. **定对象层**：明确正在解决的具体问题与约束。
2. **选抽象层**：选择能覆盖问题群的最小上层概念。
3. **双向映射**：
   - 上 -> 下：这个概念如何指导当前动作？
   - 下 -> 上：当前案例如何修正概念边界？
4. **层级校准**：
   - 过高：变正确废话，降级抽象。
   - 过低：只能解释个案，升级抽象。
5. **收口输出**：产出“原则 + 应用 + 边界”。

## 反模式

- 只谈原则不谈对象。
- 只堆案例不提共性。
- 抽象层级频繁跳跃导致对话失焦。

## 输出模板

- `object_layer`
- `concept_layer`
- `principle_to_action`
- `boundary_update`

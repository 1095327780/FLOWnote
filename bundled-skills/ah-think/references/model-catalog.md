# Model Catalog

## 选择原则

- 先按任务类型选主模型，再加一个校验模型。
- 若信息不足，优先证据类模型；若高不确定，优先校准类模型。
- 不同模型输出要能拼成“可执行动作”，否则减少模型数量。

## 模型分组与调用建议

| 分组 | 主模型 | 典型用途 | 常配校验模型 |
|---|---|---|---|
| 概念理解 | 费曼技巧、判别模型、抽象阶梯 | 把概念讲清并可判别 | 证据校准、苏格拉底提问 |
| 推理建模 | 联结模型、信息推测、第一性原理 | 从输入推到输出，重建推理链 | 贝叶斯更新、不确定性校准 |
| 决策评估 | 二阶思考、逆向思维、能力圈 | 做选择并控制风险 | 预演失败、钢人化反方 |
| 根因改进 | 5 个为什么、双环学习、杠杆点思维 | 从症状到机制，再到系统改进 | Goodhart 定律、OODA 循环 |
| 学习方法 | 经验/模型预测、渐构分析法、自下/自上学习、继承/创业学习、内隐/外显模型、记忆编码、抽象层级 | 设计学习与迁移策略 | 信息推测、证据校准 |
| 情境编排 | Cynefin、OODA 循环 | 处理动态与高不确定环境 | 不确定性校准、贝叶斯更新 |

## 快速路由表

- 观点太空泛：`feynman-technique` + `abstraction-ladder`
- 争论不收敛：`socratic-questioning` + `steelman`
- 结论不稳：`evidence-calibration` + `uncertainty-calibration`
- 决策易后悔：`second-order-thinking` + `pre-mortem`
- 学习低效率：`information-speculation` + `iterative-construction-analysis`
- 进度反复波动：`systems-leverage` + `double-loop-learning`

## 使用约束

- 单轮最多 2 个模型，避免模型堆叠。
- 如果 1 个模型已产出可执行结论，不强行加模型。
- 任意模型都必须输出“适用边界 + 下一步动作”。

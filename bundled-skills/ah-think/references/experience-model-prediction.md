# 经验预测与模型预测

## 底层原理

大脑可用两种机制做预测：

- 经验预测：基于已存样本的模式匹配与回放，优势是快、稳、低认知负担。
- 模型预测：基于变量关系的生成式推断，优势是可迁移、可解释、可外推。

它们不是对立关系，而是“默认经验、必要时建模”的协作关系。

## 适用边界

### 优先经验预测

- 场景高度重复、噪声小。
- 目标是快速识别/回忆，而不是解释机制。
- 错误代价较低，且有即时反馈修正。

### 优先模型预测

- 场景变化大、存在未见样本。
- 需要解释“为什么”，而不止“是什么”。
- 错误代价较高，需要可审计推理链。

## 详细操作流程

1. **问题定型**：这是“检索已见”还是“推断未见”？
2. **复杂度评估**：变化维度、噪声、样本覆盖度。
3. **模式选择**：经验 / 模型 / 混合。
4. **执行预测**：
   - 经验：提取最相似样本 -> 给出最近邻判断。
   - 模型：定义变量 -> 建立映射 -> 给出推断。
5. **验证回路**：对照真实结果，记录偏差来源。
6. **更新策略**：
   - 经验不足：扩充样本库。
   - 模型失配：修正变量或关系假设。

## 反模式

- 对高变化任务仍死用经验预测。
- 对简单重复任务过度建模，拖慢决策。
- 把“记住例子”误当“掌握模型”。

## 质量检查

- 我是否明确说明了为何选经验或模型？
- 该预测是否能处理一个未见新样本？
- 我是否定义了预测失败时的更新动作？

## 输出模板

- `prediction_mode`: experience / model / hybrid
- `selection_rationale`
- `validation_plan`
- `update_rule`

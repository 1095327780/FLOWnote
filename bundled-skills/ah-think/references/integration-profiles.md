# Integration Profiles

## Common Anti-Rigid Rules

- 每轮默认 2-4 个问题后先总结，不连续追问。
- 若用户已给出可执行结论，立即收口，不追求“问满配额”。
- 若用户明显疲劳，降级到最小输出（结论 + 1 个动作）。
- 同一会话不重复同一问题句式超过 2 次。

## mode=card (for ah-card)

目标：把候选洞见打磨为可制卡结论。

### 操作序列

1. 澄清断言：一句话说明“这张卡到底在说什么”。
2. 校准边界：在哪些条件成立/不成立。
3. 验证迁移：能否跨项目或跨场景复用。
4. 连接落点：至少 2 条可连接知识线索。

### 停止条件

- 已能写出断言式标题 + 边界 + 连接线索。
- 或 `confidence=low` 且信息不足，转“暂不制卡”。

## mode=read (for ah-read)

目标：从阅读内容提炼“值得制卡”的洞见。

### 操作序列

1. 提炼变化：这段内容改变了什么看法。
2. 形成机制：为什么这个结论成立。
3. 检查迁移：未来 3-6 个月能否落地。

### 停止条件

- `card_readiness=high`：可交接 `ah-card`。
- `card_readiness=medium`：保留候选，待更多证据。
- `card_readiness=low`：留在文献笔记。

## mode=review (for ah-review)

目标：把反思转成可执行改进行动。

### 操作序列

1. 找偏差：今天最关键偏差是什么。
2. 分层归因：动作层问题还是假设层问题。
3. 设动作：明天最小可执行改进动作。
4. 设护栏：如何提前发现再次偏离。

### 停止条件

- 已产出“1 个动作 + 1 条护栏”。

## mode=independent

目标：独立完成问题分析与决策建议。

### 操作序列

按 `adaptive-orchestration.md` 动态选包执行。

### 停止条件

- 输出满足 `thinking-output-schema.md` 必填字段。

# 双环学习 (Double-Loop Learning)

## 底层原理

单环学习修正动作，双环学习修正动作背后的目标、规则与假设。

若只做单环，系统会“更努力地重复同样错误”。

## 适用场景

- 同类问题反复发生。
- 执行很努力但结果始终不稳。
- 组织规则与现实冲突。

## 不适用 / 降级条件

- 一次性偶发错误可先做单环修复，再观察是否需要升双环。

## 详细操作流程

1. **事件复盘**：记录目标、动作、结果、偏差。
2. **单环修正**：先给出动作层修复（流程、协同、节奏）。
3. **假设审计**：追问“我们为什么这样设目标/规则”。
4. **识别失配**：找出导致偏差的核心假设或激励机制。
5. **双环重构**：调整目标函数、规则、评价指标。
6. **实验验证**：在小范围验证新规则有效性。
7. **制度化沉淀**：把有效规则写入常规流程。

## 与 FLOW 的结合

- `ah-read`：识别“技巧层建议”背后的前提假设。
- `ah-card`：记录“动作修正 + 假设修正”双层卡片。
- `ah-review`：把复盘升级到规则层而非仅执行层。

## 常见误用与纠偏

- 误用：复盘只停在“下次更认真”。
  纠偏：必须回答“现有规则为什么会导致该结果”。
- 误用：直接改制度但无验证。
  纠偏：先小范围试验再推广。

## 质量检查

- 是否区分了单环问题与双环问题？
- 是否明确了要调整的假设或规则？
- 是否设置了新规则的验证机制？

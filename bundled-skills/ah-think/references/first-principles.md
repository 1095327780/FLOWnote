# 第一性原理 (First Principles Thinking)

## 底层原理

第一性原理的目标是把问题从“行业习惯”还原到“不可绕开的事实约束”，再从事实重新组合方案。

它解决的是“类比路径依赖”问题，而不是否定经验本身。

## 适用场景

- 现有方案成本高、效率低、长期无改进。
- 团队理由主要是“大家都这么做”。
- 需要在约束内做结构性创新。

## 不适用 / 降级条件

- 低风险重复任务，直接沿用成熟流程更划算。
- 时间极紧时，先用类比法交付，再补第一性重构。

## 详细操作流程

1. **定义目标函数**：明确你优化什么（质量/速度/成本/风险）。
2. **列出现有假设**：把“默认正确”的规则全部写出。
3. **拆分为基础约束**：区分物理约束、逻辑约束、制度约束。
4. **逐条验真**：每条假设标注“事实/经验/惯例”。
5. **重建方案空间**：只保留硬约束，重新组合可行路径。
6. **设计最小验证**：用低成本实验验证关键新假设。
7. **回收更新**：把实验结果写入新规则，沉淀为可复用方法。

可选追问（按需）：
- 这条约束是不可违反，还是我们习惯不去碰？
- 若把成本砍半，必须改变哪条前提？
- 什么证据会推翻当前方案？

## 与 FLOW 的结合

- `ah-read`：识别“概念新颖但未落地”的内容，补足约束映射。
- `ah-card`：形成“核心假设 -> 约束 -> 新方案”结构卡片。
- `ah-review`：把失败归因从“执行差”推进到“假设错”。

## 常见误用与纠偏

- 误用：把“常识反对”误当“第一性思考”。
  纠偏：必须写出可验证的基础约束。
- 误用：每件小事都从零推导。
  纠偏：只在高价值瓶颈处使用。
- 误用：停留在抽象口号。
  纠偏：要求最小实验和量化指标。

## 质量检查

- 是否区分了硬约束与习惯约束？
- 是否提出了可实验验证的新组合？
- 是否有明确触发条件决定继续/回滚？

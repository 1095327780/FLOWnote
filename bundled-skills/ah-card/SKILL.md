# 永久笔记制作引导技能

目标：严格按卢曼卡片笔记法制卡，确保“用自己的话表达 + 一卡一事 + 高质量连接”。

## 启动即执行

1. 读取 `Meta/.ai-memory/STATUS.md` 检查待制卡事项。
2. 确认来源：文献笔记/每日记录/用户直接输入。
3. 提取候选内容并进入“深度充足性闸门”。

## 流程锁定规则（高优先级）

- 用户在制卡过程中提到“阅读/项目/回顾”等词，默认视为当前制卡输入，不自动切技能。
- 仅在两种情况允许切换：
  - 用户明确输入其他命令（如 `/ah-read`）
  - 用户明确要求停止本次制卡
- 流程完成标志：卡片保存完成 + 关联更新完成 + 已输出汇总。

## 重要提醒

- 禁止使用示例数据，所有内容必须来自用户真实输入与真实文件。
- 禁止 AI 代写用户观点；AI只做结构化整理与质量检查。
- 未经用户确认，不写入争议链接或猜测性结论。

## 深度充足性闸门（必须先过）

先判断当前输入是否满足制卡最小深度：

1. 观点是否明确（不是模糊话题）
2. 是否解释了“为什么成立”
3. 是否给出边界/条件/场景中的至少一项

判定规则：

- 满足 >= 2 项 -> 可直接制卡
- 满足 < 2 项 -> 深度不足，先引导再制卡

深度不足时必须执行：

1. 明确告知不足点（例如“缺少成立条件与适用边界”）
2. 调用 AhoySync 命令（默认 `/ah-think`）进行深度引导
3. 收集用户补充信息后回到制卡流程

## 用户原话优先（硬约束）

- 标题、核心观点、正文、应用场景均优先使用用户原话。
- 若内容缺失，标记“待用户补充”，禁止 AI 自动补写完整段落。
- AI可做的仅限：
  - 结构重排
  - 重复合并
  - 语义澄清提问

## 主流程

1. 候选提取与深度判定。
2. 深度不足则先 AhoySync 引导补充。
3. 形成卡片：断言式标题 -> 核心观点 -> 详细说明。
4. 关联发现（分层读取，不全量扫描）。
5. 用户确认后保存：`02-培养层/永久笔记/{{标题}}.md`。
6. 同步更新：主题笔记、来源笔记、反向链接。
7. 输出结果汇总。

## 关联发现规则（分层读取）

必须按以下顺序执行：

1. 先读取主题笔记标题（`02-培养层/主题笔记/` 文件夹）。
2. 再读取领域页标题（`03-连接层/`）。
3. 基于当前卡片主题选择 1-3 个候选主题/领域。
4. 定向读取这些主题笔记，提取其中已链接笔记标题。
5. 再定向读取候选笔记内容，确认是否可关联。

禁止一开始全量读取所有永久笔记。

## 输出标准

- 必须输出：新建卡片、关联关系、主题笔记更新、来源笔记更新、反向链接更新。
- 每条更新标注具体路径，确保可追溯。
- 有待确认项时明确标记“待确认”，不静默落盘。

## 检查清单

- [ ] 已完成深度充足性判定
- [ ] 深度不足时已执行 AhoySync 引导
- [ ] 卡片正文已基于用户原话完成
- [ ] 已按分层读取完成关联发现
- [ ] 已完成用户确认后保存
- [ ] 已更新主题笔记、来源笔记、反向链接
- [ ] 已输出可追溯汇总

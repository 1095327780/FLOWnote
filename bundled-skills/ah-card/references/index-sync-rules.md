# Card Index Sync Rules

## 目标

任何新卡片创建后，必须同步到：
- 主题页（📍，培养层）
- 领域页（🌱，连接层）

## 目录约定

- 主题页主路径：`02-培养层/📍*.md`
- 主题页兼容路径：`02-培养层/永久笔记/📍*.md`
- 领域页：`03-连接层/🌱*.md`

## 同步输入

每张新卡至少提供：
- 卡片标题与路径
- 标签与关键词
- 来源类型（read/review/project/archive）
- 若有项目来源：项目总览或项目编号

## 强制流程

1. 调用 `ah-index(sync-card)`，并显式传入 `sync_mode=strict`。
2. 先检索主题页候选（关键词/标签/主题 hints）。
3. 再检索领域页候选（项目语境/行动语境/领域 hints）。
4. 按固定分节写入（由 `ah-index` 执行）：
   - 主题页：`## 📚 核心知识` 下的匹配子分节（或 `### 新增卡片`）
   - 领域页：`## 🧠 核心支撑知识` 下 `### 关键永久笔记`
5. 接收返回结果并写回状态：成功页、跳过页、未命中项。

## 未命中处理

若某类页面未命中：
- 保留卡片创建结果。
- 生成待同步项并写 `待交接:ah-index`。
- 由 `ah-index` 在后续巡检中给出新建页建议。

## 同步质量要求

- 不重复插入同一链接（全页去重）。
- 不重复创建同名标题（分节去重）。
- 不修改目标页原有结构，仅做增量更新。
- 返回“主题页命中数 + 领域页命中数 + 待同步数”。

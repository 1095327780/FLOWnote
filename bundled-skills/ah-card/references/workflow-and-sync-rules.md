# ah-card 详细流程与同步规则

## 1) 候选提取

来源可为：

- 文献笔记
- 每日记录
- 用户直接输入

对来源内容先抽取候选洞见，再做深度判定。

## 2) 深度充足性判定

每条候选检查三项：

1. 观点是否明确
2. 是否解释“为什么成立”
3. 是否提供边界/条件/场景（至少一项）

判定：

- >= 2 项满足 -> 可制卡
- < 2 项满足 -> 深度不足，先引导

## 3) 深度不足处理（AhoySync）

当深度不足时，必须执行：

1. 告知不足点（具体到缺失维度）
2. 调用 AhoySync 命令（默认 `/ah-think`）
3. 收集用户补充后重新判定
4. 通过后继续制卡

示例提示：

`当前内容观点清楚，但缺少成立条件与适用边界。先用 /ah-think 做一轮深挖，再继续制卡。`

## 4) 制卡步骤

1. 标题：断言式完整句。
2. 核心观点：一句话。
3. 详细说明：200-400 字。
4. 关联关系：2-3 条（支持/反对/补充）。
5. 用户确认：标题、正文、关联。

## 5) 用户原话约束

- 标题和正文尽量使用用户原话。
- AI 不得编造新观点或新例子补全卡片。
- 若信息不够，写“待用户补充”，不自动补写。

## 6) 关联发现（分层读取，禁止全量扫描）

按顺序执行：

1. 读取主题笔记标题（`02-培养层/主题笔记/` 文件夹）。
2. 读取领域页标题（`03-连接层/`）。
3. 选择与当前候选最相关的 1-3 个主题/领域。
4. 读取这些主题笔记内容，提取其中现有链接笔记标题。
5. 再读取候选笔记正文，确认能否建立关联。
6. 输出建议关联并说明关系类型。

禁止：

- 一开始遍历并读取所有永久笔记
- 未验证正文就直接建立关联

## 7) 保存与同步顺序（不可打乱）

1. 保存永久笔记
2. 更新主题笔记
3. 更新来源笔记（衍生卡片清单）
4. 更新反向链接
5. 输出汇总

## 8) 失败恢复

若任一步骤失败：

1. 报告失败位置
2. 停止后续写入
3. 修复后从失败步骤继续

禁止静默失败。

# 项目创建自动化技能

## ⚠️ 重要提醒

**本文档中的项目名称都是示例！** 如"26-01 内容更新规划"、"26-02 B站视频制作"等，这些都是**虚构的示例**，用于说明编号格式。

创建新项目时：
1. **必须先扫描实际的项目目录**：`ls "04-创造层/Projects/"`
2. **根据实际存在的项目编号**确定下一个编号
3. **永远不要假设现有项目的名称或编号**

## 概述

本技能帮助用户一键创建完整的项目结构，省去手动查找编号、创建文件夹、编写模板等繁琐步骤。

**核心价值**：
- 自动检测现有项目编号，确保编号连续
- 一键创建项目文件夹和标准结构
- 自动生成项目主页，填充基本信息
- **自动关联所属领域，更新领域页**

## 项目定义

> **项目 = 有明确目标 + 有截止日期的任务**

| 是项目 | 不是项目 |
|--------|----------|
| 完成《效率脑科学》系列视频（3月底） | 持续学习心理学 |
| 开发阅读方法论小程序（Q1） | 维护个人品牌 |
| 写一篇公众号文章（本周） | 保持健康 |

## 工作流程

```
1. 收集项目信息（AI询问）
   ↓
2. 检查现有编号（AI自动）
   ↓
3. 确定所属领域（AI询问）
   ↓
4. 创建项目文件夹（AI自动）
   ↓
5. 生成项目主页（AI自动）
   ↓
6. 更新领域页（AI自动）
   ↓
7. 反馈创建结果（AI汇报）
```

## 阶段一：收集项目信息

### 1.1 询问必要信息

```
让我们来创建一个新项目！请告诉我以下信息：

1. **项目名称**：这个项目叫什么？
   （例如：视频脚本创作、个人网站开发）

2. **项目目标**：用一句话描述这个项目要达成的目标

3. **截止日期**（可选）：预计什么时候完成？

4. **优先级**：高🔴 / 中🟡 / 低🟢
```

### 1.2 确定所属领域

```
这个项目属于哪个领域？（可多选）

1. 🌱 **内容创作** — 视频、文章、公众号等内容产出
2. 🌱 **产品开发** — 软件、工具、小程序等产品开发
3. 🌱 **个人成长** — 学习、能力提升、习惯养成
4. 🌱 **效率工具** — 方法论、流程、工具优化

请输入数字（如：1,2 表示同时属于内容创作和产品开发）
```

### 1.3 信息确认

收集完信息后，向用户确认：

```
确认项目信息：

📁 **项目名称**：{{项目名称}}
🎯 **项目目标**：{{目标描述}}
📅 **截止日期**：{{日期或"未设定"}}
⚡ **优先级**：{{🔴高/🟡中/🟢低}}
🌱 **所属领域**：{{领域1}}、{{领域2}}

确认无误吗？我将创建项目并更新相关领域页。
```

## 阶段二：检查现有编号

### 2.1 扫描目录

AI 需要读取 `04-创造层/Projects/` 下的现有文件夹，提取编号：

```bash
# 读取 04-创造层/Projects/ 下的文件夹
# 提取形如 "26-01", "26-02" 的编号
# 确定下一个可用编号
```

### 2.2 编号规则

- 项目文件夹格式：`YY-序号 项目名称`（如 26-01、26-02）
- YY 为年份后两位（2026年为26）
- 序号为两位数字，顺序递增
- 编号与项目类型无关，统一顺序编号

### 2.3 编号示例

> ⚠️ **以下仅为编号格式示例**，项目名称都是虚构的，实际编号需要扫描用户的真实目录

```
04-创造层/Projects/
├── 26-01 {{项目A}}/       ← 示例格式：YY-序号 项目名称
├── 26-02 {{项目B}}/
├── 26-03 {{项目C}}/
...
└── 26-XX {{现有最后一个项目}}/
    ↓ 新项目编号 = 现有最大编号 + 1
└── 26-YY {{新项目名称}}/
```

**实际操作时**：先执行 `ls "04-创造层/Projects/"` 获取真实的项目列表和编号

## 阶段三：创建项目文件夹

### 3.1 文件夹结构

```
04-创造层/Projects/{{编号}} {{项目名称}}/
└── 📍 项目总览.md    # 项目主页
```

### 3.2 创建操作

使用 Bash 命令创建文件夹：

```bash
mkdir -p "04-创造层/Projects/{{编号}} {{项目名称}}"
```

### 4.1 项目主页模板

3. 保存到 `04-创造层/Projects/{{编号}} {{项目名称}}/📍 项目总览.md`

使用 Write 工具写入项目主页。

## 阶段五：更新领域页

### 5.1 更新规则

根据用户选择的领域，在对应领域页的「关联项目」区块添加新项目：

```markdown
## 🚀 关联项目

### 活跃项目

| 项目 | 产出形式 | 核心知识 |
|------|----------|----------|
| [[26-10 {{项目名称}}/📍 项目总览\|{{编号}} {{项目名称}}]] | {{产出形式}} | 待确定 |
```

### 5.2 领域页位置

- `03-连接层/🌱 内容创作.md`
- `03-连接层/🌱 产品开发.md`
- `03-连接层/🌱 个人成长.md`
- `03-连接层/🌱 效率工具.md`

## 注意事项

1. **编号检查**：务必先扫描目录，确保编号不重复
2. **年份处理**：如果跨年，编号重新从 01 开始（如 27-01）
3. **领域关联**：一个项目可以属于多个领域
4. **双向链接**：创建项目后更新领域页，保持双向链接
5. **路径处理**：注意中文路径和空格的处理

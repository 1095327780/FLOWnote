# Project Gates (O -> W)

用于判断“是否值得立项”，避免把普通待办误升级为项目。

## Gate 1: Scope Gate

满足任意 2 项即可立项：

- 跨越多步且预计持续 >= 3 天。
- 需要跨知识主题协同（不只是单点执行）。
- 有明确产出物（可演示/可交付/可上线）。

不满足时：

- 不创建项目。
- 建议转普通任务或放入今日日记。

## Gate 2: Outcome Gate

立项前必须有：

- 一句话项目目标。
- 可验证成功标准（至少 2 条）。
- 预计完成时间。

缺失时：

- 写 `阻塞:<缺失字段>`。
- 返回最小补齐提问。

## Gate 3: Execution Gate

启动即要求最小可执行切片：

- 首批行动项 <= 5 条。
- 第一里程碑可在 7 天内检验。

## Gate 4: Archive Readiness Gate

立项时预置归档字段，确保后续可沉淀：

- 决策记录位。
- 经验沉淀位。
- 归档触发条件位（何时调用 `ah-archive`）。

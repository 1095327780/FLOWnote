# Daily Routing: Project Note vs Knowledge Card

用于把每日执行中产生的思考，路由到正确位置。

## Route A: 项目笔记（优先）

写入 `01-执行记录/执行日志.md`，当内容属于：

- 当天执行事实（做了什么、遇到什么问题）。
- 项目内决策（为什么这么选）。
- 实验结果（尝试 -> 结果 -> 下一步）。

## Route B: 永久卡片候选

写入 `04-沉淀候选/洞见候选.md` 并交接 `ah-card`，当内容属于：

- 可迁移到其他项目/领域的方法或原则。
- 能解释更多现象的稳定结论。
- 明确可连接已有知识网络的洞见。

## Route C: 双写策略

如果一条内容既是执行证据又有迁移价值：

1. 先写项目执行日志（保留上下文）。
2. 再写洞见候选，并标记 `待交接:ah-card`。

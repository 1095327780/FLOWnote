# Project Domain Sync Rules

## 目标

新项目创建完成后，必须同步到至少 1 个领域页（🌱），确保连接层立刻可见该项目入口。

## 同步对象

- 领域页路径：`03-连接层/🌱*.md`
- 项目入口：`04-创造层/Projects/**/📍 项目总览.md`

## 同步输入

- `project_path`
- `project_title`
- `project_id`
- `domain`
- 可选：`domain_hints`

## 执行流程

1. 确认项目目录与 `📍 项目总览.md` 已创建。
2. 基于 `domain/domain_hints` 检索领域页候选。
3. 调用 `ah-index(sync-project)`，并显式传入 `sync_mode=strict`。
4. 固定写入 `## 🚀 关联项目` 下的 `### 活跃项目` 分节。
5. 返回同步结果：命中页、跳过页、未命中页。

## 强制规则

- 未执行同步前，不算完整立项。
- 同步失败不回滚项目目录，但必须写 `待交接:ah-index`。
- 允许创建“待同步清单”，但不得忽略领域页同步动作。

## 幂等规则

- 领域页已存在同一项目链接时，跳过而不重复插入。
- 已存在同名分节时禁止重复创建，必须复用既有分节。
- 仅做增量写入，不重排页面结构。

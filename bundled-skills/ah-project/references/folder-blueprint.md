# Project Folder Blueprint (Knowledge-First)

目标：支持知识沉淀，不做复杂 PM 系统。

## Canonical Structure

```text
{{项目编号}}-{{项目名}}/
├── 📍 项目总览.md
├── 01-执行记录/
│   └── 执行日志.md
├── 02-产出物/
├── 03-参考资料/
└── 04-沉淀候选/
    └── 洞见候选.md
```

## Why This Structure

- `项目总览`：项目意图、成功标准、支撑知识、归档触发条件。
- `01-执行记录`：过程证据（决策、实验、问题、结论）。
- `02-产出物`：可交付成果（文档、素材、脚本、草稿）。
- `03-参考资料`：外部输入（链接、截图、资料摘要）。
- `04-沉淀候选`：可迁移洞见，供 `ah-card` 转化。

## Simplicity Rules

- 只保留 4 个子文件夹，避免过度分层。
- 仅预置 2 个种子文件：`执行日志.md`、`洞见候选.md`。
- 其余内容按需增量创建。

# Note Field Spec

## 目的

定义今日日记字段与写入责任，保证 `ah-note`、`ah-capture`、`ah-review` 协同。

## 字段与责任

1. `今日事项`：由 `ah-note` 写入用户确认后的事项清单。
2. `今日想法`：由 `ah-capture` 追加原话与时间戳。
3. `日终回顾`：由 `ah-review` 回填进展、收获、明日草案。
4. `想法处理结果`：由 `ah-review` 标记去向（转卡/项目沉淀/任务/跳过）。

## 字段质量要求

- `今日事项` 不限条数，但必须来自用户确认。
- `今日事项` 默认未完成；仅在用户明确说明时才可勾选。
- `今日想法` 保留原话，禁止在捕获阶段过度改写。
- `想法处理结果` 每条必须有明确去向。

## 兼容约束

- 即使只执行 `ah-note`，模板也可独立使用。
- 若未调用 `ah-capture/ah-review`，对应区块保留占位不报错。
- 不再使用 `今日上下文/节奏提醒待办/工作日志` 区块。

# Status And Handoff

## 状态文件

- 路径：`Meta/.ai-memory/STATUS.md`
- 只使用允许枚举：`待开始` / `进行中(N/M)` / `待交接:<skill>` / `已完成` / `阻塞:<原因>`

## ah-note 读写范围

`ah-note` 重点操作“回顾”分区，并读取其他分区的待交接提示。

## 写入规则

1. 当天首次启动：在“回顾”写 `{{YYYY-MM-DD}}: 待开始`。
2. 当天已启动再进入：不重复新建，保留或更新已有状态。
3. 状态非法时：纠正为允许值并记录一次修复。
4. 若启动被阻塞：写 `阻塞:<原因>`。

## 交接提示规则

- 只做提示，不在 `ah-note` 内执行交接业务。
- 若“卡片笔记/项目/阅读整理”存在 `待交接:*`，可在返回中提醒优先级。
- 下游执行由对应技能完成（如 `ah-card`、`ah-archive`）。

## 返回约束

返回最多 3 条下一步建议：
- 白天捕获：`ah-capture`
- 晚间回顾：`ah-review`
- 若有紧急交接：对应目标 skill

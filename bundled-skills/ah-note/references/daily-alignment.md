# Daily Alignment

## 目标

让今日事项与近期节奏连续，但不把日记变成项目管理面板。

## 对齐顺序

1. 用户当下口述事项（最高优先）。
2. 昨日日记未完成事项（建议池）。
3. 本周回顾未完成事项（建议池）。
4. 在途项目当日关键动作（可选建议）。

## 每日对齐规则

- 建议只能“建议”，不能未经确认直接落笔。
- 用户确认后再写入今日事项清单。
- 若用户补充新事项，以用户补充为准。
- 不单独写“今日上下文”区块；对齐信息体现在事项本身。

## 建议池最小写法

- 昨日遗留：`[ ] {{事项}}（来自昨日）`
- 本周遗留：`[ ] {{事项}}（来自本周）`
- 项目事项：`[ ] {{事项}} @{{项目}}`

## 反模式

- 直接覆盖用户原始计划。
- 计划条目硬性限制为 3 条。
- 为了“完整”而填充与当天无关的背景信息。

# Startup Principles

## 底层原理

日启动的目标是让用户在最短时间进入执行，而不是把模板写复杂。

每日笔记只保留三件核心事：
- 今天要做什么（事项清单）；
- 今天出现了什么想法（想法捕获）；
- 今天结束后如何复盘（日终回顾）。

## 什么时候用

- 每天开工前。
- 工作中断后重启节奏时。
- 需要把零散任务重新拉回可执行状态时。

## 详细操作流程

1. 先记录用户明确提出的事项（不限制条数）。
2. 再提取昨日/本周未完成项，作为“建议池”。
3. 由用户决定建议池保留哪些，再写入今日日记。
4. 预留“今日想法”区，供白天快速捕获。
5. 预留“日终回顾”区，供晚间闭环。

## 反模式

- 强制收敛为单一 MIT，丢失用户真实计划。
- 在未确认前自动写入或自动勾选事项。
- 混入上下文、节奏提醒、工作日志等冗余区块。

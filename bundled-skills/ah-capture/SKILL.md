# 快速捕获助手技能

## 概述

知识管理的第一步是捕获。本技能帮助用户以最低摩擦将想法**直接记录到今日日记**中。

**核心价值**：
- 降低记录门槛，让捕获变得无痛
- **保留用户原话**，不过度加工
- 所有想法集中在一处，便于回顾
- 与每日工作流无缝整合

## 捕获原则

> **先捕获，后整理。宁可多记，不可漏记。**

- 每日笔记是当天所有想法的汇集地
- 记录当下的想法，不要纠结格式
- 晚上复盘时（`/ah-review`）会自动回顾这些想法

## ⭐ 核心原则：保留用户原话

**AI 的职责边界**：
- ✅ 添加时间戳
- ✅ 简单的格式整理（去掉语气词）
- ❌ **不要改写用户的表达**
- ❌ **不要美化或润色内容**
- ❌ **不要添加用户没说的内容**

### 输入方式处理规则

**语音输入**（语音转文字）：
- 只去掉无意义的语气词：嗯、啊、那个、这个、就是说、然后呢
- 保留用户的原始表达和用词
- 不改变句子结构

**打字输入**：
- **原封不动记录**
- 不做任何修改

## 工作流程

```
1. 接收用户输入
   ↓
2. 检查/创建今日日记（如不存在则先创建）
   ↓
3. 追加到日记的「💡 想法和灵感」区域
   ↓
4. 反馈完成
```

`01-捕获层/每日笔记/{{YYYY-MM-DD}}.md`

如果今日日记不存在，先调用 `ah-note` 的逻辑创建。

## 追加格式

在每日笔记的 `### 💡 想法和灵感` 区域追加：

```markdown
- {{HH:mm}} {{用户的内容}} {{#标签（如有）}}
```

**示例**：
```markdown
### 💡 想法和灵感

- 09:15 习惯的本质是大脑的自动化程序，改变习惯要替换行为而非消除触发
- 10:30 用户不会读说明书，界面要自解释 #产品
- 14:22 复杂系统的关键不是功能多，而是反馈快 #产品思维
```

## 注意事项

1. **低摩擦**：不要问太多问题，快速记录是第一要务
2. **保留原话**：语音输入只去语气词，打字输入原封不动
3. **时间戳**：自动添加记录时间，方便后续回顾
4. **不做判断**：先记录，好坏在晚上复盘时再说
5. **今日日记优先**：如果日记不存在，自动创建后再记录

## 与每日工作流的关系

```
早上 ☀️ /ah-note  →  白天 💡 /ah-capture  →  晚上 🌙 /ah-review
    创建日记              随时记录想法            回顾想法，决定去向
                              ↓
                        追加到今日日记
                        「💡 想法和灵感」区域
```

**相关技能**：
- **ah-note**：创建今日日记
- **ah-review**：晚上回顾时处理当天的想法
- **ah-card**：将有价值的想法转为永久笔记

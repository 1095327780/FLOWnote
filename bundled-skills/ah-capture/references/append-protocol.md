# Append Protocol

## 目标

统一 `ah-capture` 写入格式，确保 `ah-review` 可逐条处理。

## 写入目标（双通道）

1. 想法通道：`## 今日想法（ah-capture 追加）`
2. 事项通道：`## 今日事项（用户确认后写入）`（仅进展/完成更新）

若目标区块不存在，应自动补齐标题后再写入。

## 想法通道格式

```markdown
- [ ] {{HH:mm}} {{想法原话}} {{可选标记}}
```

## 事项通道格式

- 明确“完成”：匹配到事项后标记 `[x]`，并在该事项后追加进展说明。
- 明确“进行中/阻塞”：保持 `[ ]`，在该事项后追加进展说明。

进展说明示例：

```markdown
- [ ] 测试 ah-capture 捕获功能
  - 14:30 进展：已完成主要流程测试，待补边界用例
```

```markdown
- [x] 测试 ah-note 创建/续写功能
  - 16:10 更新：用户确认“已完成”
```

## 匹配规则

1. 优先按事项关键词做最近似匹配。
2. 匹配不确定时，不自动改事项状态，改写入想法通道并提示待确认。
3. 禁止凭推断把事项改为完成。

## 禁止项

- 删除原句核心含义。
- 在捕获阶段重写成结论或卡片体。
- 把“进展反馈”仅写到想法区而不更新事项区（可更新时）。

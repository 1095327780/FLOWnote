# ah-capture 路由判定与边界场景

## 1) 路由判定顺序

按顺序匹配，命中即执行，不继续向后判定：

1. 记录必写：任何输入先写入 `## 记录` 一条（单段整合格式）。
2. 任务完成附加动作：包含“完成/做完/已完成/搞定”等完成语义。
3. 明日计划附加动作：包含“明天/明日/明早 + 待办语义”。
4. 新增任务附加动作：包含“加任务/待办/提醒/需要做/还要做”等任务语义。
5. 批量捕获：多行列表、连续短句、明显并列输入（逐条执行“记录必写”，再逐条判断附加动作）。
6. 默认：仅保留记录写入，无附加动作。

## 2) 单句歧义处理

仅在以下情况追问：

- 需要附加动作但语义冲突（例如同时出现“完成”与“新增”）
- 目标对象不清楚（例如“这个做完了”但无可匹配任务）

追问模板（只能一句）：

`这是要标记任务完成、作为今日任务，还是放到明日计划？`

## 3) 链接场景处理

用户输入包含 URL 时：

1. 必须调用 OpenCode 内置网络工具解析链接主要内容（`websearch` + `webfetch`）。
2. 解析成功：在记录条目末尾追加 `（链接摘要: ...）`。
3. 解析失败：不追加摘要也可，但原始链接必须保留。
4. 禁止把摘要拆成多行单独块，必须与原文整合在同一条记录段落中。

## 4) 跨技能关键词处理

用户输入中出现“阅读/项目/卡片/复盘”等词时：

- 默认仍按捕获处理，不自动切技能。
- 可在确认后附加轻提示：`晚上可用 /ah-review 或 /ah-card 继续处理。`

## 5) 批量输入处理

批量输入示例：

```
- 想法A
- 想法B
- 想法C
```

处理规则：

- 同一批次统一时间戳（分钟级）。
- 每一条都必须写入 `## 记录`（记录必写）。
- 若用户明确某条是明日事项，单条附加路由到 `## 明日计划`。
- 若用户明确某条是今日任务，单条附加路由到 `## 任务`。
- 某条含链接时，仅对该条执行链接解析，结果仍整合在该条记录里。

## 6) 交互模式结束词

识别以下词结束本轮：

- 完成
- 结束
- 没有了
- 就这些

结束后输出本轮汇总，不继续追问。

# ah-capture 写入与更新规则

## 1) 今日日记文件

目标文件：`01-捕获层/每日笔记/{{YYYY-MM-DD}}.md`

约束：捕获与补充内容统一落在当日日记（`## 记录` / `## 任务` / `## 明日计划`）。

若不存在：

1. 创建最小兼容结构：

```markdown
# {{YYYY-MM-DD}} 星期X

## 今天最重要的事
写今天唯一最重要的一件事，要求可执行且可在今天推进。

## 任务
写今天任务清单，第一条对齐“今天最重要的事”。

## 记录
写白天发生的重要进展、想法或事件。

## 晚间回顾
写做得好的地方、可改进处和明天最想推进的一件事。

## 明日计划
写明天预计推进的 1-3 件重要事项，供次日承接。
```

2. 再执行写入。

若已存在但为旧版区块（如 `## 🌅 今日计划`、`## 📝 今日记录`），先补齐标准五区块再写入。

## 2) 记录区写入（记录必写）

写入到 `## 记录`：

```markdown
- {{HH:mm}} {{单段整合内容}}
```

规则：

- 用户发送任何想法，必须先写入 `## 记录` 一条，不得漏记。
- 语音输入仅去除语气词（嗯、啊、那个）。
- 保留原话，不重写观点。
- 若含 URL，原始链接必须原样保留（包括顺序与文本形态，不删不改）。

## 3) 链接解析与单段整合

仅在输入包含 `http://` 或 `https://` 链接时执行：

1. 抽取链接并按出现顺序去重。
2. 调用 OpenCode 内置网络工具解析主要内容（`websearch` + `webfetch`）。
3. 将解析结果附加到同一条记录末尾，形成单段整合文本。

格式：

```markdown
- {{HH:mm}} {{用户原话（含原始链接）}}（链接摘要: {{域名A}}: {{摘要A}}; {{域名B}}: {{摘要B}}）
```

规则：

- 每个链接摘要建议不超过 50 字。
- 链接解析失败时可不附摘要，但不得编造内容。
- 禁止拆成独立多行摘要（如 `> 📎 ...`），避免与时间戳分离导致内容混乱。
- 多链接场景下必须保留全部原始链接。

## 4) 新任务写入（附加动作）

写入到 `## 任务`：

```markdown
- [ ] {{任务内容}}
```

规则：

- 去掉“帮我加个任务”等提示词，仅保留任务正文。
- 不擅自补截止日期。
- 该动作是附加动作，不替代 `## 记录` 的必写条目。

## 5) 明日计划写入（附加动作）

写入到 `## 明日计划`：

```markdown
- [ ] {{任务内容}}
```

规则：

- 仅在用户明确“明天/明日/明早”语义时写入该区块。
- 与 `## 任务` 分开维护，避免混入今日执行清单。
- 该动作是附加动作，不替代 `## 记录` 的必写条目。

## 6) 任务完成更新（附加动作）

在 `## 任务` 中匹配待办项并更新：

```markdown
- [ ] 需求文档初稿
```

更新为：

```markdown
- [x] 需求文档初稿
```

匹配策略：

1. 精确匹配
2. 包含匹配
3. 语义近似匹配

若有多个候选，列出序号让用户确认。

若无候选，追问：

`没找到对应任务，要把这条作为已完成记录写到“记录区”吗？`

## 7) 写入后校验

每次写入后必须校验：

- `## 记录` 条目已落盘（必检项）
- 含链接时原始链接仍存在于记录条目中
- 若有附加动作，其目标区块写入已成功
- 返回内容与写入内容一致

## 8) 错误恢复

若写入失败：

1. 重读文件并重试一次
2. 仍失败则报告失败原因
3. 不做静默失败

# Write Path Recovery

## 场景 1：今日日记不存在

处理顺序：
1. 调用 `Read ../ah-note/SKILL.md` 创建今日日记。
2. 创建后回到目标区（今日想法/今日事项）执行追加。
3. 返回“已创建并写入”结果。

## 场景 2：今日日记存在但结构不完整

处理顺序：
1. 检查是否有目标区块（今日想法/今日事项）。
2. 缺失时补齐区块标题。
3. 再执行标准追加。

## 场景 3：文件冲突或暂时不可写

处理顺序：
1. 返回阻塞原因（锁冲突/权限不足/路径异常）。
2. 给最小修复动作（关闭冲突写入、恢复权限或路径）。
3. 修复后重试写入今日日记，不创建任何临时容器。

## 场景 4：输入内容极长

处理顺序：
1. 拆分为多条可处理单元。
2. 保留原话顺序。
3. 标记 `?` 供晚间重点澄清。

## 恢复原则

- 只允许恢复到“今日日记主路径”。
- 不创建独立容器，不创建临时捕获文件。

# 阅读笔记整理

目标：把长篇划线文档拆成可控章节批次，逐步提炼洞见并沉淀为可续作的文献笔记。

## 启动即执行

1. 读取 `Meta/.ai-memory/STATUS.md` 的“阅读整理”区块。
2. 读取阅读进度文件：`Meta/.ai-memory/reading/《书名》.md`。
3. 若 `STATUS.md` 或阅读进度文件不存在，按 `ah-memory` 接口规范先创建目录与最小文件结构，再继续流程。
4. 若用户提供了微信读书导出文档路径，先执行章节索引脚本，输出全部三级标题与行号。
5. 基于脚本输出由 AI 判断“大章节”并生成章节范围统计，再让用户选择先处理哪一章。
6. 若本轮大章节数量 `<= 5`，追加输出下一层候选标题（仍由 AI 判断是否采用）。
7. 确认本书转化策略（二选一）：
   - `逐章转卡`：每章完成后可立即调用 `ah-card`
   - `全书后转卡`：先完成整本书整理，最后批量转卡
8. 若用户未明确选择转化策略，不进入后续处理，必须继续追问直到选定策略。

## 流程锁定规则（高优先级）

- 用户回答默认是阅读输入，不因出现“项目/任务/记录”等词自动切技能。
- 仅在两种情况允许切换：
  - 用户明确输入其他命令（如 `/ah-card`）
  - 用户明确要求停止本次整理
- 调用 `ah-card` 后必须返回 `ah-read`，确认是否继续当前阅读进度。
- 流程完成标志：章节索引已落盘 + 文献笔记已写入 + 进度与 `STATUS.md` 已同步。

## 重要提醒

- 禁止一次性读取整篇长文；必须按“章节行号范围”分段读取。
- 禁止使用示例数据；书名、章节、划线数量、想法数量都来自真实文档。
- 用户说“今天先到这”时先保存进度；是否询问转卡必须按当前策略执行（逐章转卡=询问，全书后转卡=不询问）。
- 用户要“做成卡片/永久笔记”时必须调用 `ah-card`，不得在本技能内替代执行。

## 触发机制

- 主动触发：用户输入“阅读整理 / 整理划线 / 读书笔记整理”。
- 路由触发：`ah` 入口识别到阅读划线处理需求时调用。

## 主流程

1. 阶段一：收集阅读上下文（书名、来源、作者、动机、整理模式、评分、标签）。
2. 阶段二：运行脚本输出三级标题行号；AI判断大章节并生成章节范围与数量统计。
3. 阶段三：按章节范围分批处理（每次只读取当前章节行范围）。
4. 阶段四：提炼洞见并更新阅读进度文件。
5. 阶段五：生成/更新文献笔记，并在文献笔记顶部写入“阅读整理进度”区块。
6. 阶段六：按转化策略执行分支（逐章转卡 / 全书后转卡）。
7. 阶段七：回写 `Meta/.ai-memory/STATUS.md` 并输出下一步建议。

## 转化策略分支（确定性）

### A. 逐章转卡

1. 每章完成后询问：是否立即调用 `ah-card` 处理本章候选洞见。
2. 若用户同意：先写回进度与候选池，再调用 `ah-card`。
3. 若用户拒绝：保留候选池，状态标记为“待后续转卡”，继续下一章。

### B. 全书后转卡

1. 处理中间章节时不重复追问“是否现在转卡”，默认继续下一章。
2. 每章完成后只更新候选池与进度，不触发 `ah-card`。
3. 当“已完成章节 == 总章节”时，统一询问是否调用 `ah-card` 进行批量转化。
4. 若用户选择稍后再转：必须在 `STATUS.md` 保留明确待办与候选数量。

## 关键执行规则（确定性）

- 章节索引脚本固定：`assets/extract_weread_outline.py`。
- 步骤 1（标题提取）命令：

```bash
python3 ".opencode/skills/ah-read/assets/extract_weread_outline.py" headings "<文档路径>"
```

- 步骤 2（范围统计）命令：

```bash
python3 ".opencode/skills/ah-read/assets/extract_weread_outline.py" ranges "<文档路径>" --chapter-lines "75,1131,1671,2191"
```

- 大章节判断由 AI 完成，优先依据：`第一章/第二章`、`第一幕/第二幕`、`场景一/场景二`、`1.1/2.3` 等模式。
- 当大章节数量 `<= 5` 时，脚本自动输出该层下一层候选标题（含行号与子章节候选标记），供 AI 二次判断。
- 分批处理时只读取章节范围对应行，避免全文上下文膨胀。
- 阅读进度文件路径固定：`Meta/.ai-memory/reading/《书名》.md`。
- 文献笔记路径固定：`02-培养层/文献笔记/《书名》.md`。
- 文献笔记 Frontmatter 字段来源固定：
  - 用户输入：来源、作者、评分、标签
  - 规则自动：创建时间（当前日期）、类型（文献笔记）、状态（阅读中/阅读完成）
- 文献笔记顶部必须包含“阅读整理进度”区块，记录：总章节、已完成章节、当前章节、下一章节、最近更新时间。
- 文献笔记必须维护“候选洞见池”区块，逐条记录：章节、洞见摘要、状态（待转卡/已转卡/放弃）。
- 阅读进度文件必须维护“转化策略”“章节进度（总章节/已完成章节）”和“卡片转化进度”（待转卡数、已转卡数、放弃数）。
- 候选洞见池的跨会话权威副本固定在阅读进度文件；文献笔记作为展示与追溯副本。
- `STATUS.md` 回写采用最小格式：
  - `《书名》：整理中 N/M（逐章转卡/全书后转卡）`
  - `《书名》：待整理卡片笔记（X条，来源：本章/全书）`
- 禁止丢候选：调用 `ah-card` 之前，候选洞见必须已写入阅读进度文件（权威副本）并同步到文献笔记；任一写入失败即停止调用。

## 暂停与续作规则

触发：用户明确“先到这/下次继续”。

执行：

1. 先保存当前章节与已提炼洞见。
2. 输出当前进度（已完成章节/总章节、已处理划线与想法数量）。
3. 根据转化策略执行：
   - 逐章转卡：询问是否立即调用 `ah-card`。
   - 全书后转卡：不追问转卡，直接确认下次从下一章节继续。
4. 若用户明确“全书读完再整理卡片”，写入阅读进度文件与 `STATUS.md` 的“全书后转卡”标记。

## 输出标准

- 必须输出：
  - 本次处理章节与行号范围
  - 本次章节统计（划线数、想法数）
  - 本次候选洞见池变化（新增/已转卡/放弃）
  - 当前转化策略（逐章转卡/全书后转卡）
  - 已写入文件路径（文献笔记 + 阅读进度 + `STATUS.md`）
  - 下一步建议（继续下一章或 `/ah-card`）
- 输出必须可追溯，禁止只说“已完成”。

# 文献笔记模板（AH Read）

## 用途

用于承接阅读整理的主输出。目标是“可复盘、可筛选、可交接”，不是原文堆叠。

## Frontmatter

```yaml
---
创建时间: {{YYYY-MM-DD}}
类型: 文献笔记
来源: 《{{书名}}》
作者: {{作者}}
状态: 阅读整理中
标签: [文献笔记, {{主题}}]
---
```

## 正文模板

```markdown
# 📖 《{{书名}}》

## 📖 基本信息
- 书名：{{书名}}
- 作者：{{作者}}
- 阅读日期：{{日期}}
- 整理深度：{{快速/标准/深度}}

## 🎯 阅读动机
{{本次阅读要解决的问题}}

## 🧠 核心概念（我的解释）
- {{概念1}}：{{用自己的话解释机制}}
- {{概念2}}：{{用自己的话解释机制}}

## 💡 洞见时刻
### {{断言式标题1}}
- 触发摘录："{{原文或批注}}"
- 我的理解：{{为什么重要}}
- 适用边界：{{何时成立/不成立}}
- 路由建议：{{留文献/待交接ah-card/项目沉淀}}

### {{断言式标题2}}
- 触发摘录："{{原文或批注}}"
- 我的理解：{{为什么重要}}
- 适用边界：{{何时成立/不成立}}
- 路由建议：{{留文献/待交接ah-card/项目沉淀}}

## 🚀 行动灵感
- {{可执行动作1（最小实验）}}
- {{可执行动作2（可选）}}

## 📚 划线整理（按主题）
### {{主题A}}
- {{摘录1}}
- {{摘录2}}

### {{主题B}}
- {{摘录1}}
- {{摘录2}}

## 🔗 待交接候选（可空）
- [ ] {{候选标题}} — {{一句话理由}}（`待交接:ah-card`）

#文献笔记
```

## 书写原则

- 允许保留关键原文，但每条洞见必须有“我的理解”。
- 优先少量高质量洞见，不追求条目数量。
- 不满足转卡门槛的内容留在文献笔记，不强转。

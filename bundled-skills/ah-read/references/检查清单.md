# 检查清单（每次执行必看）

## 进度与文件
- [ ] 是否检查了进度文件？
- [ ] 是否确认了本书转化策略（逐章转卡/全书后转卡）？
- [ ] 是否执行了三级标题行号提取脚本？
- [ ] 是否完成了大章节判断并生成章节范围统计？
- [ ] 划线超过20条时，是否先分类了？
- [ ] 分类后，是否创建了进度文件？
- [ ] 每批次完成后，是否更新了进度文件？
- [ ] 调用 `ah-card` 前，候选洞见是否已写入阅读进度文件（权威副本）并同步到文献笔记？
- [ ] 最后是否用 Write 工具写入了文献笔记？
- [ ] 文献笔记顶部是否包含“阅读整理进度”区块？
- [ ] 文献笔记是否更新了“候选洞见池（转卡状态）”？
- [ ] 是否确认用户能看到生成的文件？
- [ ] 是否已回写 `Meta/.ai-memory/STATUS.md`（阅读整理与卡片笔记区块）？
- [ ] 若为“全书后转卡且暂不转卡”，`STATUS.md` 是否写入待转卡数量与来源（本章/全书）？

## 文献笔记 Frontmatter
- [ ] 是否补齐字段：创建时间、类型、来源、作者、状态、评分、标签？
- [ ] 是否按规则写入固定字段（创建时间/类型/状态）？

## 卡片笔记引导（关键！）
- [ ] 用户说"今天先到这"时，是否按当前策略执行（逐章转卡=询问；全书后转卡=仅保存进度）？
- [ ] 整理完成时，是否按当前策略询问转卡（逐章转卡=本章后询问；全书后转卡=全书完成后统一询问）？
- [ ] 用户说要整理卡片笔记时，是否调用了 ah-card 技能？（不能自己处理！）
- [ ] 是否读取了 `ah-card/SKILL.md` 并按其流程执行？

## 策略分支一致性
- [ ] 若用户未明确策略，是否继续追问直到选定（未选定不得继续处理）？
- [ ] 若策略为“逐章转卡”，本章结束后是否询问转卡并按需调用 `ah-card`？
- [ ] 若策略为“全书后转卡”，中间章节是否跳过转卡追问并继续下一章？
- [ ] 若“已完成章节 == 总章节”，是否统一询问一次批量转卡？

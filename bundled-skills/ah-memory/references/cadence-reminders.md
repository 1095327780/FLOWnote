# Cadence Reminders (Weekend/Month-End)

## 目标

把周回顾与月回顾提醒嵌入到每日流程（`ah-note` / `ah-review`），避免“只做日闭环，不做周月闭环”。

## 触发窗口

- 周回顾提醒窗口：周六、周日。
- 月回顾提醒窗口：当月最后两天（`月末-1` 与 `月末`）。

## 注入规则

1. 触发窗口首日（周六或月末-1）：
   - 询问是否将回顾任务加入今日日记待办。
   - 用户同意则立即写入待办并标记“提醒注入已完成”。
   - 用户拒绝则不写入，但保留“提醒注入待开始”，下一触发日继续检查。
2. 触发窗口末日（周日或月末）：
   - 若仍未注入，必须自动写入待办（强制链路）。
3. 若本周期回顾已完成或待办已存在，不再重复提醒。

## 待办文案约定

- 周回顾：`[ ] 本周回顾（ah-week）`
- 月回顾：`[ ] 本月回顾（ah-month）`

## 状态写法（回顾分区）

允许使用以下键名，状态值仍必须是标准枚举：

- `{{YYYY-Www}} 周回顾执行: <状态>`
- `{{YYYY-Www}} 周回顾提醒注入: <状态>`
- `{{YYYY-MM}} 月回顾执行: <状态>`
- `{{YYYY-MM}} 月回顾提醒注入: <状态>`

## 幂等约束

- 写入前先检查今日日记是否已有同名待办；已有则跳过并标记注入完成。
- 同一周期不得重复插入同一提醒任务。

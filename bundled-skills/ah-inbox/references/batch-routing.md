# Batch Routing

## Route A：项目执行笔记

适用：执行事实、项目决策、实验结果、项目风险。

写入：项目 `01-执行记录/执行日志.md`

## Route B：卡片候选（交接 ah-card）

适用：可迁移、可断言、可连接的稳定洞见。

写入：
- 项目场景可先写 `04-沉淀候选/洞见候选.md`
- `STATUS.md` 卡片笔记分区写 `待交接:ah-card`

## Route C：任务

适用：需要行动但不形成知识结论的条目。

写入：下一日计划或任务清单。

## Route D：跳过 / 已处理

适用：低价值、重复、已失时效。

写入：处理结果并给简短原因。

## Route E：待澄清

适用：上下文不足或去向不明。

动作：调用 `ah-think`；仍不清晰则保留待澄清标签。

## 分流顺序

1. 先判是否项目特定（是则优先 A）。
2. 再判是否满足转卡门槛（满足则 B）。
3. 再判是否可执行任务（C）。
4. 否则 D 或 E。

# 想法整理助手技能

## 🔒 流程锁定规则（最高优先级）

> ⚠️ **进入本技能后，必须完成完整流程才能退出！**

**核心原则**：用户的回答是「数据输入」，不是「命令触发」。

**禁止跳转的情况**：
- 展示想法列表时，用户说「第3条是关于阅读的想法」
  - ❌ 错误：识别到「阅读」跳转到 ah-read
  - ✅ 正确：继续处理该条想法
- 用户选择「转为任务」并说「这是个项目想法」
  - ❌ 错误：识别到「项目」跳转到 ah-project
  - ✅ 正确：将想法转为任务

**调用 ah-card 后必须返回**：
- 选择「转为永久笔记」时会调用 ah-card
- ah-card 完成后，**必须返回 ah-inbox 继续处理下一条想法**
- 直到所有想法处理完毕

**流程完成标志**：
- 所有想法都已处理（或用户选择跳过剩余）
- 显示了处理结果汇总

## 概述

每日笔记中会积累大量想法和灵感。本技能帮助用户定期（建议每周）清理这些内容，将有价值的想法转化为永久笔记。

**核心价值**：
- 自动扫描近期日记，提取待处理的想法
- 帮助用户快速决策每条内容的去向
- 将有价值的洞察转化为永久笔记
- 保持每日笔记的整洁

## 处理原则

**定期整理**：
- 建议每周末使用一次
- 或者在 `/ah-week` 周回顾时自动提醒

**三种去向**：
1. **转永久笔记**：有价值的洞察、可复用的方法
2. **转任务**：需要行动的事项
3. **标记已处理**：不需要进一步处理，但保留在日记中作为记录

## 工作流程

```
1. 扫描近期每日笔记（默认7天）
   ↓
2. 提取「💡 想法和灵感」区域的内容
   ↓
3. 展示内容概览
   ↓
4. 逐条处理（AI引导+用户决策）
   ↓
5. 执行操作
   ↓
6. 总结汇报
```

## 阶段一：扫描每日笔记

### 1.1 定位日记文件

每日笔记路径：`01-捕获层/每日笔记/`

扫描范围：默认过去7天，可由用户指定

### 1.2 提取想法

从每个日记文件的 `### 💡 想法和灵感` 区域提取内容。

**提取规则**：
- 识别以 `- ` 开头的条目
- 提取时间戳（如有）
- 提取标签（如有）
- 过滤已标记为「✅ 已处理」的条目

### 1.3 展示概览

```
📝 想法整理开始！

扫描了过去 **7** 天的每日笔记，发现 **{{N}}** 条待处理的想法：

| # | 日期 | 时间 | 内容预览 | 标签 |
|---|------|------|----------|------|
| 1 | 02-03 | 09:15 | 习惯的本质是... | |
| 2 | 02-03 | 14:22 | 用户体验的核心... | #产品 |
| 3 | 02-04 | 10:30 | 复杂系统的关键... | #产品思维 |
| ... | ... | ... | ... | ... |

**按日期分布**：
- 02-03：3条
- 02-04：2条
- 02-05：1条

准备开始处理吗？我会逐条帮你决定去向。
```

## 阶段二：逐条处理

### 2.1 展示单条内容

```
📝 **第 {{i}}/{{N}} 条**

**日期**：{{YYYY-MM-DD}}
**时间**：{{HH:mm}}
{{#标签（如有）}}

---
{{完整内容}}
---

这条想法应该：
1. 💡 转为永久笔记（有价值的洞察）
2. ✅ 转为任务（需要行动）
3. ☑️ 标记已处理（保留但不需进一步处理）
4. ⏭️ 跳过（稍后处理）

你的选择是？
```

### 2.2 根据选择处理

#### 选择1：转为永久笔记

调用 `/ah-card` 的流程：

```
好的，让我们把它转化为永久笔记。

这条想法的核心观点是什么？
（或者我来帮你概括：{{AI建议的概括}}）

永久笔记标题建议：
- {{用陈述句表达的标题}}

相关主题：
- [[02-培养层/📍 {{主题}}]]

确认创建吗？
```

#### 选择2：转为任务

```
这是一个需要行动的事项。

我会把它添加到今日日记的任务列表中：
- [ ] {{任务内容}}

确认添加吗？
```

#### 选择3：标记已处理

```
好的，这条想法会标记为已处理，保留在原日记中。
```

在原日记中将该条目改为：
```markdown
- ~~09:15 习惯的本质是...~~ ✅ 已处理
```

#### 选择4：跳过

```
好的，跳过这条。下次继续处理。
```

### 2.3 批量处理模式

如果想法很多，提供批量模式：

```
想法较多（{{N}}条），要使用批量模式吗？

批量模式下，我会先帮你分类，然后你批量确认：
- 建议转永久笔记：{{n}}条
- 建议转任务：{{n}}条
- 可能不需要处理：{{n}}条
- 需要你判断：{{n}}条
```

## 阶段三：执行操作

### 3.1 创建永久笔记

> 🔗 **必须调用 `ah-card` 技能**：不要直接使用模板创建，而是调用永久笔记制作技能。

调用 ah-card 时传入：
- 原始想法内容
- 原始日期和时间
- 用户确认的核心观点

ah-card 会负责：
- 引导确定断言式标题
- 扫描知识库推荐链接
- 归类到相关主题页
- 维护双向链接

### 3.2 更新原日记

将已处理的想法标记：
```markdown
- ~~09:15 原始内容~~ ✅ → [[永久笔记标题]]
```

或任务：
```markdown
- ~~09:15 原始内容~~ ✅ → 已添加到任务
```

## 注意事项

1. **尊重用户判断**：AI 只是建议，最终决策权在用户
2. **保留原始记录**：想法在日记中标记为已处理，但不删除
3. **快速节奏**：避免每条内容讨论太久
4. **批量模式**：内容多时提供批量处理选项
5. **周期提醒**：建议在周回顾时使用

# Reflection Loop

## 事项复盘三问

1. 今天计划事项完成了哪些？
2. 未完成项的真实原因是什么？
3. 明天最先推进的事项是什么？

## 标准操作流程

1. 复述“今日事项”清单，不新增虚构事项。
2. 逐项判断：完成 / 部分完成 / 未完成。
3. 为未完成项写原因（执行问题 / 估时问题 / 外部约束）。
4. 提炼 1-2 条可复用收获。
5. 生成明日事项草案（可多条，需有先后顺序）。
6. 回填到 `## 日终回顾（ah-review 回填）`。

## 深度模式补充

- 对每个关键事项再问：
  - 这件事背后的假设是什么？
  - 下次如何更早识别风险？
  - 哪一步可固化为可复用方法？

## 最小回顾模式（quick）

- 至少写：完成项、未完成原因、明日首项。
- 状态写 `进行中(N/M)`，待次日补完想法处理。
